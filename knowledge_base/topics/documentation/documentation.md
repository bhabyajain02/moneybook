## Core Technical Reference Content
*   A complete technical reference document should cover:
    *   Product overview and problem statement.
    *   Full system architecture and tech stack.
    *   SQL schema and directory structure.
    *   AI architecture (model selection, token economics).
    *   Deep-dives into core code files, including E2E flow traces.
    *   API calls with parameters and retry logic.
    *   Annotated prompts for AI components.
    *   Known bugs with root causes and fixes.
    *   Performance characteristics and deployment guide.
    *   Future roadmap.
